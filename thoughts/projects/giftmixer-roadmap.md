# GiftMixer — Roadmap & Vision

## Где мы сейчас

Продукт работает. Основные пайплайны (крафты, стикер-паки, cinema, постеры, анимации) в production. 57 edge functions, 51 стиль, 15 агентных скиллов. Freemium экономика прописана, security закрыт.

**Что уже даёт ценность:**
- Пользователь берёт свой подарок → за 10 сек получает стикер-пак
- Создаёт видео-историю из персонажей с озвучкой
- Делает крафт (fusion) двух подарков в нового
- Всё это внутри Telegram, без выхода из мессенджера

## Фаза 1 — Стабилизация (сейчас)

**Цель:** Каждая существующая фича работает надёжно, без багов, с хорошим UX.

| Задача | Приоритет | Статус |
|--------|-----------|--------|
| Deploy generate-lottie-structure | HIGH (блокер) | Нужен деплой |
| Character Sheet Generator (frame extraction) | HIGH | Планируется |
| Scene transitions (crossfade/morph) | HIGH | Планируется |
| Clean dead code in smart-prompt-engine | MEDIUM | ~1700 → <1600 строк |
| Cover auto-generation для стикер-паков | LOW | Nice-to-have |

## Фаза 2 — Умный роутер

**Цель:** Пользователь описывает что хочет → система сама выбирает инструмент.

Сейчас пользователь должен знать разницу между "крафт", "стикер-пак", "cinema". В идеале: пишет "хочу видео где мой Skibidi танцует" → система сама определяет что это Cinema Studio → Kling video → export.

**Архитектура:**
```
User text → Gemini 3.1 (intent parsing) → Tool selection → Params → Execute
```

Не нужен полный оркестратор. Просто LLM-прослойка которая парсит intent и маршрутизирует.

## Фаза 3 — Pipeline Composer

**Цель:** Цепочки из 2-3 шагов с checkpoint UX.

Самый востребованный сценарий: персонаж → видео → lipsync → export.

```
Step 1: Generate character image (Gemini Pro) → [User: OK/Retry]
Step 2: Animate character (Kling 3.0) → [User: OK/Retry]
Step 3: Add voice (ElevenLabs + Lipsync) → [User: OK/Retry]
Step 4: Export → Telegram/TikTok/Stories
```

Каждый шаг имеет цену, пользователь видит total cost до старта.

## Фаза 4 — Полный оркестратор

**Цель:** Произвольные workflow из любого намерения.

LLM читает Tool Registry → строит DAG → исполняет. Новые модели добавляются без кода — только запись в реестр.

## Фаза 5 — Платформа

**Цель:** Выход за пределы подарков.

- Генерация контента из любых изображений (не только подарки)
- TikTok/Reels формат: склейка клипов, тренды
- Creator economy: топ-создатели зарабатывают на установках их паков
- Виральный цикл: создал → поделился → друг установил → создал своё

## Нарратив: откуда и куда

```
Эксперимент: "а что если стикеры из подарков?"
    ↓
Оказалось: owned assets = эмоциональная привязанность
    ↓
Добавили больше инструментов (крафты, видео, постеры)
    ↓
Поняли: это не набор инструментов, это Intent Platform
    ↓
Пользователь описывает что хочет → AI делает
    ↓
Каждый новый AI-инструмент = новая возможность без нового UI
    ↓
Tool Registry как живой документ
    ↓
Flywheel: подписка → эксперименты → лучшие результаты → шеринг → рост
```

**Конечное видение:** Telegram-нативная AI-платформа где любой контент создаётся через описание намерения. Подарки — точка входа и эмоциональный якорь. Со временем — любые изображения, видео, audio.

## Конкурентное преимущество

1. **Owned assets** — подарки принадлежат пользователю, эмоциональная привязанность
2. **Telegram-native** — не нужно скачивать app, всё внутри мессенджера
3. **Intent-driven** — не нужно разбираться в AI, просто опиши что хочешь
4. **Агентная разработка** — 15 специализированных скиллов = скорость итераций

## Метрики успеха

- **North Star:** Monthly Active Creators (создали хотя бы 1 контент)
- **Engagement:** Sessions per creator per week
- **Monetization:** Revenue per creator, conversion free → paid
- **Virality:** k-factor (сколько новых юзеров приводит каждый)
